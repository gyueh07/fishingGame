# FishingLife UI 개편판

기존 텍스트 낚시게임의 저장 데이터와 게임 규칙을 유지하면서 화면과 파일 구조를 개편한 버전입니다.

## 이번 버전에서 바뀐 점

- 모바일과 PC에 맞춰 자동으로 배치가 바뀌는 반응형 게임 UI
- 낚시터, 양동이, 도감, 보스전, 성장, 광장 화면
- 물고기별 이미지를 사용하지 않고 종류별 아이콘과 등급 효과를 조합한 카드 UI
- 기존 명령어와 상세 전투 로그를 사용할 수 있는 게임 기록 패널
- 모든 보스의 일반·어려움·크레이지 난이도와 난이도별 최초 상금
- 코어 보상: 일반 1~3개, 어려움 3~5개, 크레이지 5~10개
- 공허 등급 보스 에레보스·크로노스·니알라토텝·요그 소토스·아자토스 추가
- 크레이지 전용 궁극기와 난이도별 강화 패시브
- 크라켄·베히모스·수르트·케르베로스·아지다하카·크로노스·니알라토텝·아자토스의 크레이지 진화 패시브
- 보스 공격·일반 스킬·패시브 전환·크레이지 궁극기를 차례대로 보여주는 실시간형 전투 재생 화면
- 보스와 파티 체력 변화, 턴 표시, 공격 중인 물고기 강조, 1·2·4배속, 건너뛰기, 다시보기 지원
- 전투 하단의 작은 로그 카드 제거 및 1배속을 기존의 3분의 1 속도로 조정
- 보스 난이도 선택 화면에서 전투력·공격력·체력·등급·최근 획득 순 정렬
- 일반·어려움·크레이지 쿨타임을 따로 적용하고 선택 화면에서 남은 시간이 실시간으로 감소
- 보스전 중 최대 체력 감소 효과가 훈련소 동기화에 의해 완전 회복되던 문제 수정
- 요르문간드 크레이지 공격력 증가 한도 150%가 실제 피해 계산에도 적용되도록 수정
- 캐스팅·입질·PERFECT·전설 이상 획득·보스 공격·치명타·승리 효과음
- 효과음과 모바일 진동을 각각 켜고 끌 수 있는 기기별 설정
- 보스 어려움 최초 클리어 시 전용 프로필 테두리, 크레이지 최초 클리어 시 전용 오라 해금
- 보스 테두리 장착 시 기본 닻 대신 해당 보스 문양을 표시
- 같은 등급 보스의 크레이지를 전부 클리어하면 등급 전용 배경과 한정 공격 연출을 동시에 해금
- 영웅·전설·신화·초월·영원·공허 등급마다 서로 다른 배경과 공격 연출
- 영웅은 청색, 공허는 보라·암적색 계열로 분리해 등급 색상을 명확하게 구분
- 낚싯대 버튼 아래의 타이밍 미니게임 시작 문구 제거
- 원본 물고기 능력치, 별, 보스 수치와 전투 패턴 유지
- 훈련소 구간별 누적 성장: Lv.1~5 단계당 +20%, Lv.6~10 +50%, Lv.11~15 +100%
- 기존 양동이 물고기에도 상향된 훈련소 배율 즉시 적용
- PERFECT, GREAT, GOOD, BAD 낚시 타이밍 미니게임과 확률 보정
- 타이밍 구간을 노랑·초록·파랑·빨강의 단색으로 구분하고 판정은 낚아채는 순간에만 표시
- GREAT는 기존 낚싯대 확률, PERFECT는 소폭 보너스, GOOD·BAD는 단계별 페널티 적용
- BAD는 일반·희귀 비중이 크게 증가하고 초월 이상 확률과 도주 성공률에 강한 페널티 적용
- 최근 획득한 물고기 10마리 표시 및 클릭 상세정보
- 양동이의 180마리 표시 제한을 제거해 보유 물고기 전체 표시
- 양동이 카드와 상세정보에 기존 공격력·체력·회피·치명타·치명타 피해 1~3별 표시 복원
- 보스전 재생에서도 양동이와 동일한 물고기별 이모지 사용
- 물고기 도주 시 낚시 상태를 즉시 종료하고 이름·등급·판정·도주 확률을 전용 화면과 문구로 표시
- 낚시터를 다시 열 때 이전 획득 연출이 반복되던 문제 수정
- 물고기 이름에 맞는 전용 이모지 표시
- 기존 1~1750레벨 성능을 유지한 낚싯대 2000레벨 확장
- 1750~2000레벨에 공허 보스 5종의 코어 재료 연결
- 채팅형 명령어 화면을 제거하고 시세·도감·보스전·PVP를 게임형 화면으로 표시

## 파일 구조

```text
index.html                 화면 구조
css/legacy.css             기존 로그·모달 기본 스타일
css/game-ui.css            새 반응형 게임 UI
js/data/fish-data.js       물고기, 등급, 크기, 특성 데이터
js/data/combat-data.js     전투 범위, 보스, 코어 데이터
js/core.js                 계정, 저장, 공통 상태, 연구·훈련
js/combat.js               보스전과 PVP 전투
js/gameplay.js             낚시, 경제, 양동이, 도감 등
js/commands.js             기존 명령어 연결과 시작 처리
js/ui.js                   새 화면 표시와 버튼 연결
```

## 배포

프로젝트의 파일과 폴더 구조를 유지한 채 정적 웹 호스팅의 루트에 업로드합니다. 기존 게임과 같은 Firebase 프로젝트를 사용하므로 Firebase 설정과 허용 도메인은 기존 배포 환경을 그대로 사용합니다.

## 저장 데이터 호환

전투 능력치 버전은 원본과 동일하게 유지했습니다. 저장된 물고기의 `combat` 데이터가 다시 생성되거나 약해지지 않습니다.
